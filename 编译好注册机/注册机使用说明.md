# 注册机（bl_keygen）使用说明 / 註冊機使用說明 / Keygen User Guide

本文档说明 **`bl_keygen.exe`** 的用途与操作步骤。注册机**仅供软件发行方在己方电脑使用**，切勿分发给最终客户。

---

## 一、简体中文

### 1. 程序作用

- 生成 **RSA 密钥对**（`private.blb` 私钥、`public.blb` 公钥）。
- 根据顾客提供的 **机器码** 等信息，生成 **Base64 授权码**，顾客将授权码粘贴到 **加壳后的程序** 授权窗口中完成激活。

### 2. 首次使用：生成密钥对

1. 将 **`bl_keygen.exe`** 放在任意文件夹（建议单独建目录，仅你本人可访问）。
2. 运行 **`bl_keygen.exe`**。
3. 点击 **「生成密钥对（首次）」**。
4. 同目录下会出现：
   - **`private.blb`**：**私钥，严格保密**，用于签发授权；**不要**发给任何人，**不要**打进给客户用的程序包。
   - **`public.blb`**：**公钥**，用于 **加壳工具** 写入客户程序，或与壳程序同目录分发（视你采用的发布方式而定）。

若需更换密钥：可再次点击「生成密钥对」，**旧私钥签发的授权码将失效**，已发客户需重新签发。

### 3. 给顾客签发授权码

1. 顾客运行你提供的 **加壳后的 exe**，窗口中会显示 **机器码**（一串十六进制字符）。
2. 顾客将机器码发给你。
3. 在注册机中填写：
   - **顾客机器码**：粘贴顾客发来的内容（须完全一致）。
   - **产品名**：与签发时约定一致（默认示例为 `myapp`，须与授权逻辑中使用的名称一致）。
   - **到期 Unix 时间（秒）**：例如 `1893456000` 表示该时间戳对应的 UTC 时刻到期；可用在线工具将日期转换为 Unix 秒。
4. 点击 **「生成授权码」**，下方文本框会出现 **一长串 Base64**。
5. 将 **整段授权码** 复制发给顾客；顾客粘贴到授权窗口并点击验证启动。

### 4. 不绑定单台机器（慎用）

在 **顾客机器码** 一栏填写 **`*`**（星号），可生成 **任意机器** 均可使用的授权码。仅建议在内部测试或合同另有约定时使用，避免滥用导致盗版扩散。

### 5. 与加壳工具的配合

- 使用 **`bl_packer.exe`** 对客户程序加壳时，需选择本目录下的 **`public.blb`**（与当前密钥对匹配）。
- **私钥** 只留在注册机所在环境，**不要**随加壳输出或客户包一起发布。

### 6. 常见问题

| 现象 | 处理 |
|------|------|
| 提示无法加载 private.blb | 确认已先生成密钥对，且 exe 与 private.blb 同目录。 |
| 顾客提示授权失败 | 核对机器码是否抄错、产品名是否与签发时一致、是否已过期、公钥是否与签发私钥为同一对。 |
| 更换电脑使用注册机 | 将 **private.blb** 与 **bl_keygen.exe** 一并拷贝到新机器（注意安全）。 |

## 二、繁體中文

### 1. 程式用途

- 產生 **RSA 金鑰對**（`private.blb` 私鑰、`public.blb` 公鑰）。
- 依顧客提供的 **機器碼** 等資料，產生 **Base64 授權碼**；顧客將授權碼貼到 **加殼後的程式** 授權視窗以完成啟用。

### 2. 首次使用：產生金鑰對

1. 將 **`bl_keygen.exe`** 置於任意資料夾（建議獨立目錄，僅本人可存取）。
2. 執行 **`bl_keygen.exe`**。
3. 點選 **「生成密钥对（首次）」**（介面用語與簡體版相同）。
4. 同目錄將出現：
   - **`private.blb`**：**私鑰，務必保密**，用於簽發授權；**切勿**提供予他人，**切勿**一併封裝給客戶程式。
   - **`public.blb`**：**公鑰**，供 **加殼工具** 寫入客戶程式，或與殼程式同目錄發行（視您的發行方式而定）。

若需更換金鑰：可再次產生金鑰對，**舊私鑰簽發之授權將失效**，已發放客戶需重新簽發。

### 3. 為顧客簽發授權碼

1. 顧客執行您提供的 **加殼後 exe**，視窗會顯示 **機器碼**。
2. 顧客將機器碼傳送給您。
3. 於註冊機填寫：
   - **顧客機器碼**：貼上顧客提供內容（須完全一致）。
   - **產品名**：與約定一致（預設範例 `myapp`）。
   - **到期 Unix 時間（秒）**：例如 `1893456000`；可用線上工具將日期轉為 Unix 秒。
4. 點選 **「生成授权码」**，下方會出現 **Base64 字串**。
5. 將 **完整授權碼** 複製給顧客；顧客貼至授權視窗並完成驗證。

### 4. 不綁定單機（請謹慎）

於 **顧客機器碼** 欄位填 **`*`**，可產生 **任意機器** 皆可使用之授權碼。建議僅內部測試或合約允許時使用。

### 5. 與加殼工具

- 使用 **`bl_packer.exe`** 時，請選擇與目前金鑰對相符之 **`public.blb`**。
- **私鑰** 僅保留於註冊機環境，**勿**隨客戶套件流出。

### 6. 常見狀況

| 狀況 | 處理 |
|------|------|
| 無法載入 private.blb | 確認已產生金鑰對，且 exe 與 private.blb 同目錄。 |
| 顧客授權失敗 | 檢查機器碼、產品名、是否過期、公鑰與私鑰是否同一對。 |

## 三、English

### 1. What this tool is

- Generates an **RSA key pair**: `private.blb` (secret signing key) and `public.blb` (public key for your protected apps).
- Creates a **Base64 license token** from the customer’s **machine fingerprint** and your settings. The customer pastes this token into the **authorization window** of your packed application.

**`bl_keygen.exe` is for you (the publisher) only.** Do **not** ship it or `private.blb` to customers.

### 2. First-time setup: generate keys

1. Place **`bl_keygen.exe`** in a folder only you can access.
2. Run **`bl_keygen.exe`**.
3. Click **「生成密钥对（首次）」** (Generate key pair – first time).
4. Files appear next to the exe:
   - **`private.blb`**: **Keep secret.** Used to sign licenses. **Never** distribute.
   - **`public.blb`**: Used by **`bl_packer.exe`** (or placed beside the launcher) to verify licenses on the customer side.

If you generate a **new** key pair, licenses signed with the **old** private key will **stop working**; re-issue licenses if needed.

### 3. Issuing a license to a customer

1. The customer runs your **packed exe** and sends you the **machine code** shown in the window.
2. In the keygen, fill in:
   - **Customer machine code**: paste exactly what the customer sent.
   - **Product name**: must match what you use when signing (default example: `myapp`).
   - **Expiry (Unix time, seconds)**: e.g. `1893456000`. Use any trusted converter from date/time to Unix seconds (UTC).
3. Click **「生成授权码」** (Generate license). A long **Base64** string appears in the output box.
4. Send the **full** string to the customer. They paste it into the auth dialog and complete verification.

### 4. Non–machine-bound licenses (use with care)

Enter **`*`** in the machine code field to create a license valid on **any** machine. Intended for internal testing or special contracts only.

### 5. Working with the packer

- When using **`bl_packer.exe`**, select the **`public.blb`** that belongs to the same key pair as your `private.blb`.
- Never bundle **`private.blb`** with customer releases.

### 6. Troubleshooting

| Issue | What to check |
|--------|----------------|
| Cannot load `private.blb` | Key pair not generated yet, or file not in the same folder as `bl_keygen.exe`. |
| Customer: license invalid | Wrong machine code, wrong product name, expired time, or `public.blb` does not match the signing key pair. |
